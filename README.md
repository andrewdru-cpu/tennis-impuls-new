# ЦТТ «Импульс» — сайт теннисного клуба

Премиальный сайт теннисного центра «Импульс» (Мытищи, рядом с национальным парком
«Лосиный Остров»). Next.js 15 + TypeScript + Tailwind CSS + Framer Motion.

---

## Запуск проекта локально

```bash
npm install        # один раз — установить зависимости
npm run dev        # запустить сайт: http://localhost:3000
```

Другие команды:

```bash
npm run build      # собрать продакшн-версию (проверка ошибок)
npm run start      # запустить собранную версию
npm run lint       # проверка кода линтером
```

## Как сохранять и публиковать изменения (Git)

После любой правки сайта:

```bash
git add .
git commit -m "Краткое описание, что изменили"
git push
```

Примеры хороших сообщений коммитов:

- `git commit -m "feat: добавил страницу расписания"`
- `git commit -m "fix: поправил телефон в контактах"`
- `git commit -m "style: обновил фото на главной"`

> Первый пуш на новый репозиторий: `git push -u origin main`
> (флаг `-u` нужен один раз, дальше достаточно `git push`).

### Авторизация на GitHub (через токен, не пароль)

GitHub не принимает пароль при `git push` — нужен **Personal Access Token (PAT)**.
Как его получить — см. раздел ниже «Personal Access Token». Токен вводится **один раз**
в окне авторизации (или в строке логина/пароля вместо пароля) и сохраняется системным
менеджером учётных данных Windows. **Никуда в файлы проекта токен записывать нельзя.**

## Personal Access Token (GitHub)

Пошагово, как создать токен вместо пароля:

1. Войдите на [github.com](https://github.com) под своим аккаунтом.
2. Откройте **Settings** (меню под аватаром справа вверху).
3. Внизу слева — **Developer settings**.
4. **Personal access tokens → Tokens (classic) → Generate new token (classic)**.
5. Заполните:
   - **Note**: например `ctt-impulse-site`;
   - **Expiration**: срок действия (например, 90 дней или No expiration);
   - **Scopes**: отметьте галочку **`repo`** (этого достаточно для пуша кода).
6. Нажмите **Generate token** и **скопируйте токен сразу** — он показывается один раз.
7. При первом `git push` Windows попросит логин и пароль:
   - **логин** — ваш GitHub-username;
   - **пароль** — **вставьте скопированный токен** (не пароль от аккаунта).
   Токен сохранится в «Диспетчере учётных данных Windows», и больше его вводить не нужно.

> Токен — это секрет. Не вставляйте его в код, в файлы проекта или в сообщения коммитов.
> Если токен утёк — удалите его на той же странице и создайте новый.

## Деплой на Vercel

Vercel — бесплатный хостинг, идеально подходит для Next.js. После настройки **каждый
`git push` в ветку `main` автоматически обновляет сайт в интернете**.

### Что уже подготовлено в проекте

- `vercel.json` — Next.js, `npm ci`, сборка `npm run build`, регион **fra1** (Франкфурт)
- `package.json` → скрипты `"build": "next build"` и `"start": "next start"`
- `.vercelignore` — тяжёлые и служебные файлы не попадают в деплой
- `.env.example` — шаблон переменных на будущее (YCLIENTS, webhook заявок)
- `next.config.mjs` — оптимизация импортов, AVIF/WebP, без лишних заголовков

> **Примечание:** режим `output: 'standalone'` **не включён** — он нужен для Docker/самостоятельного хостинга.
> На Vercel Next.js деплоится нативно, standalone только усложнит сборку.

### Шаг 1. В браузере — подключить GitHub к Vercel

1. Откройте [vercel.com](https://vercel.com) и нажмите **Sign Up** (или **Log In**).
2. Выберите **Continue with GitHub** и разрешите доступ.
3. На главной Vercel нажмите **Add New… → Project**.
4. В списке найдите репозиторий с этим сайтом и нажмите **Import**.
5. На экране настроек **ничего менять не нужно** — Vercel сам увидит Next.js:
   - **Framework Preset**: Next.js
   - **Build Command**: `npm run build`
   - **Install Command**: `npm ci`
   - **Output Directory**: (оставьте по умолчанию)
6. Нажмите **Deploy** и подождите 1–3 минуты.
7. Когда сборка станет зелёной — нажмите **Visit**. Сайт откроется по адресу вида
   `https://ваш-проект.vercel.app`.

### Шаг 2. Автодеплой при каждом push

После первого деплоя Vercel уже следит за репозиторием:

- вы меняете код локально;
- делаете `git add .` → `git commit -m "..."` → `git push`;
- Vercel **сам** запускает сборку и публикует новую версию сайта.

Статус деплоя смотрите в Vercel: **Dashboard → ваш проект → Deployments**.

### Шаг 3. Свой домен (позже, необязательно)

**Project → Settings → Domains** → добавьте свой домен (например `ctt-impulse.ru`).
Vercel покажет, какие DNS-записи прописать у регистратора.

### Переменные окружения (когда понадобятся)

Сейчас проект работает **без** переменных окружения — для деплоя ничего добавлять
не нужно. Список переменных на будущее (YCLIENTS, отправка заявок) — в файле
`.env.example` в корне проекта.

Когда интеграции появятся: ключи API и токены **не пишите в код**. Локально —
скопируйте `.env.example` в `.env.local` и заполните. В Vercel добавляйте их в:

**Project → Settings → Environment Variables**

Там же можно задать разные значения для Production / Preview / Development.

### Если деплой упал с ошибкой

1. Откройте **Deployments** → кликните на красный деплой → прочитайте **Build Logs**.
2. Локально проверьте сборку:

```bash
npm run build
```

Если `npm run build` проходит без ошибок — обычно и на Vercel всё соберётся.
Исправьте код, снова `git push` — Vercel пересоберёт автоматически.

## Как добавлять новые страницы и функции

Проект на **App Router** (папка `src/app`).

- **Новая страница**, например «Расписание» по адресу `/schedule`:
  создайте файл `src/app/schedule/page.tsx` с компонентом — маршрут появится сам.
- **Новая секция на главной**: добавьте компонент в `src/components/sections/`
  и подключите его в `src/app/page.tsx`.
- **Текст, телефон, адрес, меню**: меняются в `src/lib/site.ts`.
- **Фото и видео**: все пути собраны в `src/lib/media.ts`. Положите файл в
  `public/images/...` или `public/videos/...` и поменяйте путь — вёрстку трогать не нужно.
- **Дизайн-система** (цвета, тени, радиусы): `tailwind.config.ts` и `src/app/globals.css`.
- **Интеграция YCLIENTS / онлайн-запись**: добавляется как виджет/iframe в нужную секцию
  (например, в блок «Бронирование» — `src/components/sections/booking.tsx`).
  Ключи доступа — только через переменные окружения, не в коде.

Перед пушем полезно прогнать `npm run build` — если сборка прошла без ошибок,
значит, всё в порядке.

## Структура проекта

```
src/
  app/                  # страницы (App Router), layout, globals.css, favicon
  components/
    ui/                 # кнопки, карточки, бейджи
    media/              # MediaImage, MediaVideo (гибкие медиа-компоненты)
    hero/               # HeroMedia (фото/видео в Hero)
    sections/           # секции страницы (hero, services, booking, ...)
  lib/
    media.ts            # ⭐ все пути к фото/видео
    site.ts             # навигация, телефон, адрес, ссылка на карты
    utils.ts            # вспомогательное
public/
  images/{hero,services,team,facilities,gallery}/
  videos/               # видео Hero
  logo/                 # варианты логотипа
```
